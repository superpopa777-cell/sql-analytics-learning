# Stage 1 — Day 02  
## WHERE — фильтрация данных

WHERE используется для фильтрации строк в таблице.

Если SELECT отвечает на вопрос — что показать,  
то WHERE отвечает на вопрос — какие строки оставить.

WHERE фильтрует строки, а не столбцы.

---

## Общий шаблон SQL с WHERE

SELECT columns  
FROM table  
WHERE condition;

---

## Примеры

SELECT name, signup_date  
FROM users;

SELECT name, signup_date  
FROM users  
WHERE name = 'Alice';

---

## Базовые операторы WHERE

Равно:  
WHERE name = 'Bob';

Не равно:  
WHERE name != 'Bob';  
или  
WHERE name <> 'Bob';

Больше / меньше (часто используется с датами):  
WHERE signup_date > '2024-01-03';

---

## Фильтрация по дате

SELECT *  
FROM users  
WHERE signup_date >= '2024-01-05';

Читается так: показать пользователей, зарегистрированных с 5 января и позже.

---

## Несколько условий — AND

AND означает, что **все условия должны быть истинны**.

SELECT *  
FROM users  
WHERE signup_date >= '2024-01-01'  
AND signup_date <= '2024-01-07';

В результат попадут только пользователи, зарегистрированные в этом диапазоне дат.

---

## Несколько условий — OR и IN

OR используется, когда достаточно выполнения **хотя бы одного условия**.

IN — это удобная форма записи нескольких OR  
**для одного столбца и оператора =**.

---

### Пример OR

SELECT *  
FROM users  
WHERE name = 'Alice'  
OR name = 'Bob';

---

### Пример IN (предпочтительнее)

SELECT *  
FROM users  
WHERE name IN ('Alice', 'Bob');

Итог:  
когда используется **один столбец** и **несколько равных значений**,  
предпочтительнее использовать IN.

---

### Пример с большим количеством значений

Вариант через OR:

SELECT *  
FROM users  
WHERE name = 'Alice'  
OR name = 'Bob'  
OR name = 'Charlie';

Вариант через IN:

SELECT *  
FROM users  
WHERE name IN ('Alice', 'Bob', 'Charlie');

Итог:  
чем больше одинаковых условий для одного столбца,  
тем выгоднее и читаемее использовать IN.

---

## Когда IN использовать НЕЛЬЗЯ

Если условия относятся к **разным столбцам**  
или используются **разные операторы** (`>`, `<`, `!=`),  
оператор IN применять нельзя.

В таких случаях используется OR.

---

### Пример условий для разных столбцов

SELECT *  
FROM users  
WHERE name = 'Alice'  
OR signup_date > '2024-01-05';

Итог:  
в результат попадут пользователи с именем Alice  
**или** пользователи, зарегистрированные после 5 января.

---

## Комбинирование условий (AND + OR / IN)

SELECT *  
FROM users  
WHERE name IN ('Alice', 'Bob')  
AND signup_date > '2024-01-01';

Сначала выбираются Alice и Bob,  
затем применяется фильтрация по дате регистрации.

---

## Приоритет условий и скобки

По умолчанию AND выполняется раньше, чем OR.  
Для явного управления логикой используются скобки.

SELECT *  
FROM users  
WHERE name = 'Alice'  
OR (name = 'Bob' AND signup_date > '2024-01-05');

# Практические задания (условия)

## Задание 1 — базовая фильтрация  
**Дано:**  
Выбрать столбцы `name`, `signup_date` из таблицы `users`,  
оставив только пользователей с именем `Alice`.

---

## Задание 2 — фильтрация по дате  
**Дано:**  
Выбрать все столбцы из таблицы `users`,  
оставив пользователей, зарегистрированных **после 2024-01-05**.

---

## Задание 3 — диапазон дат (AND)  
**Дано:**  
Выбрать всех пользователей,  
зарегистрированных **не раньше 2024-01-01**  
и **не позже 2024-01-07**.

---

## Задание 4 — OR  
**Дано:**  
Выбрать пользователей с именем `Alice` **или** `Bob`.

---

## Задание 5 — IN  
**Дано:**  
Переписать задание 4, используя оператор `IN`.

---

## Задание 6 — разные столбцы  
**Дано:**  
Выбрать пользователей с именем `Alice`  
**или** зарегистрированных **после 2024-01-05**.

---

## Задание 7 — AND + IN  
**Дано:**  
Выбрать пользователей с именем `Alice` или `Bob`,  
**только если** они зарегистрированы **после 2024-01-01**.

---

## Задание 8 — приоритет условий  
**Дано:**  
Выбрать пользователей с именем `Alice`  
**или** пользователей с именем `Bob`,  
**если Bob зарегистрирован после 2024-01-05**.

---

## Задание 9 — одно значение  
**Дано:**  
Выбрать только пользователя с именем `Bob`.

---

## Задание 10 — фильтрация по дате  
**Дано:**  
Выбрать пользователей, зарегистрированных **после 2024-01-01**.

---

## Итоги Day 02

1. WHERE фильтрует строки, а не столбцы  
2. Освоен базовый синтаксис SQL с условиями  
3. Изучена фильтрация по значениям и датам  
4. Понята разница между AND (все условия) и OR (хотя бы одно)
